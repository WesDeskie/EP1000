GitHub

In this webpage, I will show you how to implement version control using GitHub and GitHub Desktop.

What is GitHub?

GitHub is a service that can be used for many things, but for me, it is most useful for version control. I can upload all the files in my webpage, and GitHub will help me keep them there, and I can edit my files on another computer, or if I was working on a group project, my teammates can edit the files on their computer too.

How do I use GitHub?

First thing you want to do is go to GitHub's website (https://github.com/). Create a GitHub account if you don't have one already, and next, create a new repository.

Take note of a few things:

  • Give you repository a name and a description
  • Remember to add a README file (This will contain information about you repository)
  • Select "MIT License"

Just like that, you've created a repository.

Next, you want to add files to your repository.

  • Click on "Add file > Upload files"
  • Select which files you want to upload
  • Click "Commit changes"

In the video above you'll notice that after uploading your files, you can double click them to open them up in GitHub. You can actually edit it straight from GitHub, but I will show you a better alternative later.

Now that you've uploaded your files, you can open you website from GitHub itself.

  • Go to "Settings > pages"
  • Select your source branch as "main"
  • Click "save"

You should see a link appear. Click on the link to open up your website, or share it with others to let them open it. Keep in mind that changes will take a while to appear in the link.
