GitHub

In this webpage, I will show you how to implement version control using GitHub and GitHub Desktop.

What is GitHub?

GitHub is a service that can be used for many things, but for me, it is most useful for version control. I can upload all the files in my webpage, and GitHub will help me keep them there, and I can edit my files on another computer, or if I was working on a group project, my teammates can edit the files on their computer too.

How do I use GitHub?

First thing you want to do is go to GitHub's website (https://github.com/). Create a GitHub account if you don't have one already, and next, create a new repository.

Take note of a few things:

  • Give you repository a name and a description
  • Remember to add a README file (This will contain information about you repository)
  • Select "MIT License"

Just like that, you've created a repository.

Next, you want to add files to your repository.

  • Click on "Add file > Upload files"
  • Select which files you want to upload
  • Click "Commit changes"

In the video above you'll notice that after uploading your files, you can double click them to open them up in GitHub. You can actually edit it straight from GitHub, but I will show you a better alternative later.

Now that you've uploaded your files, you can open you website from GitHub itself.

  • Go to "Settings > pages"
  • Select your source branch as "main"
  • Click "save"

You should see a link appear. Click on the link to open up your website, or share it with others to let them open it. Keep in mind that changes will take a while to appear in the link.

GitHub Desktop

We've created a repository, now what? We can either edit files on GitHub, which is a bit troublesome, so instead we are going to clone our repository onto our computer using GitHub Desktop.

  • Download GitHub Desktop here
  • Log into your GitHub account

First, you want to copy the repository link from GitHub like so.

This will help if you are not the creator of the repository. Next,

  • Go to "file > clone repository"
  • If you're the creator of the repository, you can select it from "GitHub.com"

If you are not the creator, you can click "URL" and paste the link you copied earlier.

You've now succesfully cloned your repository. In your File Explorer, you should see a folder named "GitHub" under "Documents". Open it, and you will see all you cloned repositories. In your repository folder, you should see the files you uploaded, as well as the README file that you created.

You can now open your files in a text editor, and edit them from there. Make some changes, and open up GitHub Desktop again. You will see what changes have been made. I made some changes to my index.html file in my demosite repository, lets see what GitHub Desktop shows.

After making changes, write a summary of the changes you made, and click "commit changes"

Push and Pull

After commiting your changes, you'll want to update the files on the GitHub website, and you do so by "Pushing" it to the origin like so:

Now, let's say you've made a change on another computer, and you want to see those changes on your home computer. You could clone the repository again, but it's easier to use the "Pull" command. I've already made some changes to my repository, lets open up GitHub Desktop again. Click on "Fetch Origin", this will fetch the files from GitHub.

Then, you can click on "Pull origin", to pull the changes onto your computer.

If you open up your files in a text editor, you'll see the changes that you've made.

Version History

Lastly, if you want to see what changes have been made previously, you can click on the "History" tab. This will show you all the changes that have been made to your file.

And that's how I used GitHub to manage my projects. Hopefully you understand how to use GitHub for version control and can implement it into you own projects!
