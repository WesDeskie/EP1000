HTML and web development

One of the most important things about prototyping is documentation. Which is why we kickstarted the module by learning the fundamentals of HTML and CSS to create the website you see in front of you right now. This was my very first website as you can probably tell by the rudimentary appearance, but then again, I learnt that when documentation is concerned, content is more critical than appearance.

One thing to note is that in this webpage, I will going through HTML. However, we were also taught another way of coding a website known as Markdown, which I go through here. I cover the basics of markdown and it's advantages and also why I chose to stick with HTML to create my webpage.

What is HTML?

HTML stands for "hyper text markup language" and in itself has no formatting. We simply use what are called tags, to create content like text, images and video etc. Therefore we have to use CSS to customise the style of our website.

  • create a text file called "index.html" (This will be your homepage)

Navigation bar

The navigation bar you see at the top of this site was created with a list. There are two types of lists, Ordered lists, and unordered lists. To make an ordered list, use the tag "<ol>" and for an unordered list, use the tag "<ul>". To add items to the list, use the tag "<li>".

Adding links

We have a navigation bar, but notice that we can't click on them and they dont lead anywhere. Therefore we have to use the "<a>" tag to add a link to our text. Create a .html text file inside your folder and we will use the "<a>" tag to link to that page. Alternatively you can add a link to another webpage online.

Images and Videos

I added some images and videos by using the "<img>" tag. This allows me to link to images and videos saved in my website folder like so.

The src is where you put your location of the file, and the alt is the alternative text that will appear should your image not be able to load. You can also adjust the width of your image by typing width = "width that you want in pixels".

Additional tags

So that's how I wrote the content for this site, using headers, paragraphs and lists. But there are also a couple of other useful tags that I used to make the site a bit easier to read. They are:

  • Horizontal rules "<hr> " (These create the lines used to seperate the sections)
  • Divisions "<div>" (These are used to divide my webpage into different parts so I can format them with CSS later)
